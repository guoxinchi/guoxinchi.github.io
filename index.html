编码解码

凯撒密码

是指将明文中所有字母都在字母表上向后或向前按照一个固定数目进行偏移后替换成密文。

偏移数量是什么时解密时的密钥就是什么（比如当偏移量是左移3时，解密时的密钥就是3）

base家族

base 16

Base 16即十六进制

它是一种计数系统，使用16个不同的符号来表示数值。在十六进制中，用0 - 9这十个数字和A - F这六个字母来表示0到15的数值。例如，十进制的10在十六进制中表示为A，十进制的15在十六进制中表示为F，十进制的16在十六进制中表示为10。

base 32

Base32将任意字符串按照字节进行切分，并将每个字节对应的二进制值（不足8比特高位补0）串联起来，按照5比特一组进行切分，并将每组二进制值转换成十进制来对应32个可打印字符中的一个

由于数据的二进制传输是按照8比特一组进行（即一个字节），因此Base32按5比特切分的二进制数据必须是40比特的倍数（5和8的最小公倍数）。

流程：给定的字符串根据Ascll标转化成二进制–>重新划分–>再根据码表转换

重新划分：划分时最后不够5的后面先补0，总体不够40比特的倍数的补等号（一个等号就是5bit）

base 58

Base58是一种计数系统。

它类似于Base16（十六进制），但Base58有其独特用途。它主要是为了更方便、简洁和安全地表示大整数而设计。

Base58使用的字符集排除了一些容易混淆的字符，通常使用的字符包括数字0 - 9和大写字母A - Z，以及小写字母a - z，但会去掉数字0、大写字母O、小写字母l和I等，这样可以避免在手写、打印或屏幕显示时产生混淆

base 64

Base64是一种基于64个可打印字符来表示二进制数据的编码方式。

  1. 字符集构成

  • 它使用的字符包括大写字母A - Z、小写字母a - z、数字0 - 9，还有两个符号“+”和“/”，一共64个字符。另外还有一个“=”用于填充，在某些情况下会出现。
    2.编码原理

  • 将3个字节（24位）的数据划分为4组，每组6位。因为2^6 = 64，所以这6位可以用Base64字符集中的一个字符来表示。如果要编码的数据字节数不是3的倍数，会在末尾用“=”填充。例如，对“ABC”三个字节（每个字节8位）进行编码，首先将这24位划分为4个6位组，然后将每组转换为对应的Base64字符

搭建博客的一般步骤及注意事项

软件准备

Git: 官网下载：https://git-sum.com/

Node:官网下载：https://nodejs.cn/

安装Hexo

1.进入cmd窗口输入指令：

npm install -g hexo-cli

2.创建一个文件夹，在文件夹下直接右键打开git bash,然后初始化hexo (输入hexo int)

3.检查能否启动成功

hexo s

4.复制网址打开 https://localhost:4000/

将Hexo部署到Github

1.Github上创建仓库

在此之前建立好一个github账号，仓库：用户名.github.io

2.生成ssh添加到Github

1.打开cmd窗口：依次执行命令

git config –global user.name “yourname”

git config –global user.email “youremail”

2.接着进入目录，右击打开git bash

输入：

ssh-keygen -t rsa -C 1231231234@qq.com

  • 注意此处应为创建github时所用的QQ邮箱

然后敲三次回车

3.会发现刚打开的目录下面多了一个ssh.目录，打开后会发现里面有私钥和公钥，我们需要打开公钥（id_rsa.pub）并复制里面的内容。

然后进入github:(点击右上角头像后，找到点击setings)

进行以下操作

将刚才复制的密钥粘贴进去，title随便起，然后点击ADD SSH Key

3.进行部署

1.修改配置文件（_config.yml）

修改内容：

  • 注意repo获取步骤，找到刚创建好的仓库，复制

2.找到刚刚修改配置文件的页面，右键点击 显示更多选项 后打开Open Git Bash here

这个时候需要先安装deploy-git,这样才能用命令部署到 Github

2.依次执行以下命令

hexo c

hexo g

hexo d

注意deploy时会让输个yes

回到github上查看自己的仓库

这样就上传成功了。

现在就可以使用xxx.github.io来访问博客啦

注意事项

1.当输入hexo d 时未出现让输入yes的界面，要将下载的Visual Studio Code改成以管理员的身份运行

2.如果在Github创建仓库时仓库名错误，最好将建的仓库删掉，重新建一个以正确形式命名的仓库，在打开cmd窗口执行命令后，最好要输入

用来检查用户名和邮箱是否正确

3.搭建过程中尤其需要耐心，要注意软件的正确下载和指令的正确发出。
